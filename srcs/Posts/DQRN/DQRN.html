AutoCar applying DQN

Qualcomm Innovation Award exhibit project. Most auto car model use CNN or RNN model because image recognition. But only DQN(Deep Q Learning - reinforce learning) can make auto car moving.

Spec

python, tensorflow, DQN, rasberry PI, RC car, AWS

Start

RC car model is this. Work place is AWS server, spec graphic card model for deep learning.

RC car

RC car run with rasberry PI, get camera information and recognition of object. Basic setting be structed formal subject, rewrite things below.

  1. Module must move server’s command, not press key or else.
    • Server send tcp packet for moving signal
    • Must guarantee order of signals.
  2. Camera information(image) fast be sent to server
    • Image data little much size, so usually use UDP , but this data must be not loose. so object detection.
    • Use XGBoost model for recognition is in rasberry PI, not server. It little heavy, but fast.
  3. TCP server construction to desktop (or AWS) Server to wifi-connect
    • Collected data have to send Desktop for deep learning.
    • Must guarantee data is right.

DQN model

First time, simulation model like atary game. Each server’s sim_models work with different parameter(threshold). 10000 * 10000 train in one time.

Default setting is n * n box, set obstacle and goal to front of car. Car go front or left, right, calculate axis. If cat get collision point minus, pass obstacle plus point, get goal plus more point.

Testing different point parameters. Check static graph & epsilon, choose best model.

Choose model test real car, check it is smooth & correct
